Detailed Description

Manages globally available application settings.

It implements the singelton pattern so that there is only one settings object throughout the application. It initially loads a file called default.php in the WEPPO directory. With loadSettings() you can load application specific settings and thereby overwrite the defaults.

WEPPO\Application\Settings::loadSettings (   $settings)

Loads a settings file and merges it with the persisting settings. This way settings can be overwritten.

The Settings can be an array or a filename of a php file relative to the APP_ROOT that will be included and should return an array with the settings.

Parameters
mixed$settingsarray or file name
Exceptions
Exceptionif settings could not be merged. eg. no array given or include file does not return an array.
