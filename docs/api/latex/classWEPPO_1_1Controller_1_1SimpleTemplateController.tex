Automated generation of parts.

The automated generation of parts is configured in the page configuration. A part needs a configuration entry {\ttfamily template.\+part.\mbox{[}part\+Name\mbox{]}} with a value like so\+: {\ttfamily \mbox{[}Template\+Class\+Name\mbox{]}\+:\mbox{[}Template\+File\+Name\mbox{]}}.

Example for part \textquotesingle{}aside\textquotesingle{}\+: 
\begin{DoxyCode}
$page->setConfig(\textcolor{stringliteral}{'part.aside'}, \textcolor{stringliteral}{'\(\backslash\)\(\backslash\)WEPPO\(\backslash\)\(\backslash\)Presentation\(\backslash\)\(\backslash\)CurlyTemplate:aside.html'})
\end{DoxyCode}


This provides a simple way to merge several parts into one page output.

If you want to pass some params to the template, you can do it this way\+: `set\+Config(\textquotesingle{}part.\+aside.\+param\+Name\textquotesingle{}, \textquotesingle{}Param\+Value\textquotesingle{})` In a P\+H\+P\+File\+Template just do this\+: `\$this-\/$>$get(\textquotesingle{}param\+Name\textquotesingle{})`
